# Как вносить правки в базу знаний

Этот документ описывает, как предложить изменения в базу знаний по кинематике. Все правки проходят проверку перед попаданием в основную версию — автор базы просматривает каждое изменение и принимает или отклоняет его.

Не нужно быть программистом. Большинство шагов делается через графические интерфейсы.

## Установка на Windows

### Шаг 1. Установить Git

1. Перейти на [git-scm.com/download/win](https://git-scm.com/download/win) — скачается автоматически
2. Запустить установщик, оставить все настройки по умолчанию, нажимать **Next** до конца
3. После установки в меню Пуск появится **Git Bash** — это и есть терминал для Git

### Шаг 2. Установить GitHub Desktop

1. Скачать с [desktop.github.com](https://desktop.github.com/)
2. Установить, запустить
3. Войти в аккаунт GitHub: **File → Options → Accounts → Sign in to GitHub.com**
4. Ввести логин и пароль GitHub

### Шаг 3. Скопировать репозиторий на компьютер

1. В GitHub Desktop: **File → Clone repository**
2. Перейти на вкладку **URL**
3. Вставить адрес: `https://github.com/PhysicsPlus/kinematics-wiki`
4. В поле **Local path** выбрать папку, куда сохранить (например, `Документы\kinematics-wiki`)
5. Нажать **Clone**

После этого на компьютере появится папка `kinematics-wiki` со всеми файлами базы.

### Шаг 4. Установить Obsidian

1. Скачать с [obsidian.md](https://obsidian.md/) → **Download for Windows**
2. Установить и запустить
3. На стартовом экране выбрать **Open folder as vault**
4. Выбрать папку `kinematics-wiki` (ту, что скачали на шаге 3)

База откроется в Obsidian — все заметки будут видны в левой панели.

## Установка на Mac

### Шаг 1. Установить Git

На Mac Git часто уже установлен. Чтобы проверить:
1. Открыть **Launchpad → Terminal** (или найти через Spotlight: Cmd+Space → «Terminal»)
2. Написать `git --version` и нажать Enter
3. Если появится версия — Git уже есть, переходите к шагу 2
4. Если нет — появится предложение установить **Command Line Tools**: нажать **Install** и подождать

Альтернатива без терминала: установить [Xcode](https://apps.apple.com/app/xcode/id497799835) из App Store — Git войдёт в комплект.

### Шаг 2. Установить GitHub Desktop

1. Скачать с [desktop.github.com](https://desktop.github.com/) — есть версия для Mac
2. Перенести в папку **Программы** (Applications)
3. Запустить, войти в аккаунт GitHub: **GitHub Desktop → Settings → Accounts → Sign in**

### Шаг 3. Скопировать репозиторий на компьютер

1. В GitHub Desktop: **File → Clone repository**
2. Перейти на вкладку **URL**
3. Вставить: `https://github.com/PhysicsPlus/kinematics-wiki`
4. Выбрать папку (например, `Документы/kinematics-wiki`)
5. Нажать **Clone**

### Шаг 4. Установить Obsidian

1. Скачать с [obsidian.md](https://obsidian.md/) → **Download for macOS**
2. Перенести в папку **Программы**
3. Запустить → **Open folder as vault** → выбрать папку `kinematics-wiki`

## Настройка Obsidian

### Установить плагин obsidian-git

Этот плагин позволяет сохранять изменения прямо из Obsidian, не открывая GitHub Desktop.

1. В Obsidian открыть **Settings** (иконка шестерёнки внизу слева)
2. Перейти в раздел **Community plugins**
3. Нажать **Turn on community plugins** (если ещё не включено)
4. Нажать **Browse**
5. В поиске написать `obsidian-git`
6. Нажать **Install**, затем **Enable**

### Проверить что всё работает

1. Нажать **Ctrl+P** (Windows) или **Cmd+P** (Mac) — откроется палитра команд
2. Написать `git` — в списке должны появиться команды вроде `Obsidian Git: Pull` и `Obsidian Git: Commit all changes`
3. Выполнить **Obsidian Git: Pull** — если ошибок нет, всё настроено правильно

## Рабочий процесс

### 1. Создать ветку для своих правок

Ветка — это отдельная «копия» базы, в которой можно работать, не затрагивая основную версию.

**В GitHub Desktop:**
1. В верхней панели найти поле **Current branch** (показывает `main`)
2. Нажать на него → **New branch**
3. Ввести название в формате `имя/тема`, например:
   - `ivan/кривошип-шатун-уточнение`
   - `maria/малые-величины-примеры`
4. Нажать **Create branch**

Теперь все изменения пойдут в эту ветку, а не в основную.

### 2. Вносить правки в Obsidian

Открыть нужную заметку и редактировать как обычный текст. Правила оформления описаны в файле `CLAUDE.md` в корне базы — там указаны требования к формулам, структуре заметок, обозначениям.

Несколько советов:
- Сохранять можно в любой момент — Obsidian сохраняет автоматически
- Формулы пишутся в LaTeX: `$формула$` для строчных, `$$формула$$` для выключных
- Если что-то непонятно в правилах — лучше уточнить, чем угадывать

### 3. Сохранить изменения (коммит)

Коммит — это «снимок» текущего состояния файлов с подписью: что изменено и зачем.

**Вариант А — через Obsidian (удобнее):**
1. Нажать **Ctrl+P** / **Cmd+P**
2. Написать `commit`
3. Выбрать **Obsidian Git: Commit all changes**
4. Ввести описание изменений, например: `Добавлен пример к задаче о маятнике`

**Вариант Б — через GitHub Desktop:**
1. В левой панели появятся изменённые файлы
2. В поле **Summary** (внизу) написать краткое описание
3. Нажать **Commit to имя-ветки**

Можно делать несколько коммитов — например, по одному на каждую заметку.

### 4. Отправить изменения на GitHub

**В GitHub Desktop:**
1. Нажать кнопку **Push origin** в верхней панели
2. Подождать несколько секунд

Теперь изменения видны на GitHub.

### 5. Создать Pull Request (запрос на проверку)

1. После push GitHub Desktop покажет кнопку **Create Pull Request** — нажать её
2. Откроется браузер со страницей создания PR
3. Заполнить описание:
   - **Title**: кратко что сделано (например, `Уточнение примеров в разделе 03`)
   - **Description**: подробнее — что изменено, почему, есть ли вопросы
4. Нажать **Create pull request**

### 6. После проверки

Автор базы просмотрит изменения. Возможны три варианта:
- **Принято** — изменения попадают в основную версию
- **Запрошены правки** — в PR появятся комментарии с просьбами уточнить что-то
- **Отклонено** — с объяснением причины

Если нужны правки — внести изменения в Obsidian, снова сделать коммит и push. PR обновится автоматически, заново создавать его не нужно.

## Правила оформления заметок

Подробные правила — в файле `CLAUDE.md`. Краткая выжимка:

**Структура заметки:**
- Аннотация в начале (1–2 предложения: о чём заметка, с чем связана)
- Блоки по уровням: `## ☆` (9 класс), `## ★` (10–11 класс), `## ⭐` (студент)
- В конце — `## Связи и продолжения`

**Формулы:**
- Дроби: только `\frac{}{}`, не `a/b` в выключных формулах
- Векторы: `\mathbf{v}` (не `\vec{v}`)
- Тригонометрия: `\operatorname{tg}`, `\operatorname{ctg}` (не `\tan`, `\cot`)
- Числа: `9{,}8` (запятая как разделитель, не точка)

**Запрещено:**
- Подзаголовки `###`
- Горизонтальные разделители `---`
- Блок `## LaTeX-код`
- Жирный шрифт для всего подряд — только для термина, вводимого впервые
