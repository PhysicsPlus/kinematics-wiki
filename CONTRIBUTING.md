# Как вносить правки в базу знаний

Этот документ описывает, как предложить изменения в базу знаний по кинематике.
Все правки проходят проверку перед попаданием в основную версию.

---

## Что понадобится (один раз)

1. **Git** — система контроля версий: [git-scm.com](https://git-scm.com/)
2. **Obsidian** — редактор базы знаний: [obsidian.md](https://obsidian.md/)
3. **Плагин obsidian-git** — для работы с Git прямо из Obsidian

### Установка obsidian-git

В Obsidian: Settings → Community plugins → Browse → найти **obsidian-git** → Install → Enable.

### Клонирование репозитория

Открыть терминал (Git Bash на Windows) и выполнить:

```
git clone https://github.com/PhysicsPlus/kinematics-wiki.git
```

Затем в Obsidian: **Open folder as vault** → выбрать папку `kinematics-wiki`.

---

## Рабочий процесс

### 1. Создать ветку для своих правок

Перед началом работы открыть терминал в папке репозитория и создать новую ветку:

```
git checkout -b имя/тема
```

Примеры названий веток:
```
ivan/криволинейное-уточнения
maria/малые-величины-примеры
```

Формат: `имя/краткое-описание` (латиницей, через дефис).

### 2. Вносить правки в Obsidian

Работать как обычно — редактировать заметки, добавлять примеры.

Правила оформления описаны в файле `CLAUDE.md`.

### 3. Сохранить изменения (коммит)

В палитре команд Obsidian (Ctrl+P) выполнить:

```
Obsidian Git: Commit all changes
```

Ввести краткое описание: что именно изменено и зачем.

Можно делать несколько коммитов по ходу работы.

### 4. Отправить изменения на GitHub

В терминале:

```
git push -u origin имя/тема
```

### 5. Создать Pull Request

На странице репозитория на GitHub появится кнопка **Compare & pull request**.

В описании Pull Request указать:
- Что изменено
- Почему (исправлена ошибка, добавлен пример, уточнена формулировка)
- Если есть вопросы или сомнения — написать их здесь

---

## После отправки

Автор базы просмотрит правки и либо примет их, либо оставит комментарии с просьбой что-то уточнить. Если нужны доработки — внести изменения в ту же ветку и снова сделать коммит и push: Pull Request обновится автоматически.
