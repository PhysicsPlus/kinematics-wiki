# Брифинг для написания заметок

Этот документ — самодостаточная инструкция. Скопируй его целиком в любую нейросеть, чтобы она могла писать заметки для базы знаний по кинематике.

---

## 1. Концепция базы

**База знаний — энциклопедия концепций, не учебник.** Учебник будет собираться из базы позже.

Стиль — как в хорошем справочнике или теормехе: точный, но подробный.

## Уровни сложности

| Уровень | Маркер | Аудитория | Математический аппарат |
|---------|--------|-----------|------------------------|
| **1 — Базовый** | `## ☆` | 9 класс | Конечные разности, геометрия, площадь. Без пределов и производных. |
| **2 — Основной** | `## ★` | 10–11 класс | Производная, интеграл, предел. |
| **3 — Продвинутый** | `## ⭐` | Олимпиадники | ОДУ, дифференциальная геометрия, строгий анализ. |

Не каждая заметка содержит все три уровня — только те, которые релевантны теме.

---

## 2. Структура заметки

```markdown
---
status: draft | review | done
tags: [тема1, тема2]
level: 1 | 2 | 3 | 1-2 | 2-3 | 1-3
sources:
  - "Автор, §X, с. XX"
latex_packages: [physics, unicode-math]
relations:
  requires:
    - имя-заметки-предшественника
  leads_to:
    - имя-заметки-продолжения
---

# Название

Аннотация: что излагается в заметке, какое место занимает в структуре базы,
с какими заметками связана логически ([[ссылки]]).

## ☆ / ## ★ / ## ⭐  Содержательные блоки по уровням
(только те уровни, которые релевантны теме)

## Связи и продолжения

Абзац, объясняющий логику переходов.

- ← [[предшественник]] — почему из той темы мы приходим к этой
- → [[продолжение]] — куда эта тема ведёт дальше
```

**Типы связей** (в `relations`):
- `requires` — нужно понимать до (предпосылка)
- `leads_to` — логически продолжается в
- `generalizes` — обобщает другую заметку
- `applies` — применяет метод из другой заметки
- `alternative` — другой подход к той же проблеме
- `used_in` — математический инструмент используется в физическом контексте

## 3. Callout-блоки

**Определения и законы:**

```markdown
> [!definition]
> **Термин (уточнение).** Формулировка определения.
> $$\text{формула}$$
> где $x$ — ..., $y$ — ...
```

**Inline-примеры:**

```markdown
> [!example] Краткий заголовок
> Формулировка: что найти или вычислить.
>
> Решение: вывод и ответ.
```

Длинные примеры → отдельный файл `examples/*/пример-*.md`, в заметке только ссылка.
Жирный текст внутри `[!example]` не используется.

---

## 4. Правила написания

- Писать подробно, с примерами — разжёвывать до мельчайших подробностей, не теряя строгости.
- Каждое нетривиальное утверждение сопровождать примером или выводом.
- Формулы без объяснения — плохой тон. Объяснять, откуда берётся каждая формула.
- Таблицы — хорошо, особенно для числовых значений и сравнений.
- Физические примеры предпочтительнее абстрактных.

---

## 5. Правила оформления формул

**Дроби:**
- Везде `\frac{}{}` — никаких косых дробей.
- Исключение: в показателях степени косая допустима: `$e^{-t^2/2}$`, `$x^{1/2}$`.
- Не использовать `\dfrac` и `\displaystyle`.

**Интегралы:**
- Пределы всегда через `\limits`: `\int\limits_a^b f\,\mathrm{d}t`.

**Выключные формулы:**
- `$$...$$` без пустых строк перед и после.

**Прочее:**
- Не обводить формулы рамкой (`\boxed{}`).
- Запятая в числах — запятая: `0{,}5`, не `0.5`.
- Единицы измерения: `9{,}8~\text{м/с}^2`.
- Тригонометрия: `\operatorname{tg}`, `\operatorname{ctg}`, `\operatorname{arctg}`, `\operatorname{arcctg}`.

---

## 6. Система обозначений

**Векторы:**

| Объект | LaTeX |
|--------|-------|
| Свободный вектор | `\mathbf{v}` |
| Направленный отрезок | `\overrightarrow{AB}` |
| Модуль | `|\mathbf{v}|` или `v` |
| Единичный вектор | `\hat{\mathbf{e}}_x` |
| Греческий вектор | `\boldsymbol{\Omega}` |

`\vec{}` не использовать.

**Производные и дифференциалы:**

| Объект | LaTeX |
|--------|-------|
| Производная по времени | `\dot{x}`, `\ddot{x}` |
| Дифференциал | `\mathrm{d}t` |
| Конечная разность | `\Delta x` |

---

## 7. Структурные запреты

- `###` подзаголовки — запрещены (вместо них — абзац с вводным предложением)
- `---` горизонтальные разделители — запрещены
- `## LaTeX-код` блок — запрещён
- Жирный шрифт — только для термина, вводимого впервые
