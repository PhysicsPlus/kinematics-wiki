## Концепция базы знаний

**База знаний — энциклопедия концепций, а не учебник.**
Учебник (пособие) будет собираться из базы позже: выбором нужных заметок, выстраиванием последовательности, добавлением педагогических переходов. Сейчас важна точность и полнота, а не доступность.

**Три уровня читателей:**

| Уровень | Аудитория | Математический аппарат |
|---------|-----------|------------------------|
| **1 — Базовый** | Школьник 9 класса | Конечные разности, геометрия, площадь под кривой. Без пределов, производных, интегралов. |
| **2 — Основной** | 10–11 класс | Производная, базовое интегрирование, предел. |
| **3 — Продвинутый** | Олимпиадники 11 класса | Уверенный анализ, простейшие ОДУ, дифференциальная геометрия. |

**Ключевой принцип:** основной текст заметки (уровень 1) объясняет понятия через **конечные разности и геометрию**, без предела и производной. Блок `## ◆` добавляет строгое определение через предел/производную для уровня 2. Блок `## ⭐` — для уровня 3.

**Структура заметки:**
- Без маркера = уровень 1 (Базовый, основной текст)
- `## ◆ ...` = уровень 2 (Основной, для знающих производную)
- `## ⭐ ...` = уровень 3 (Продвинутый)

- Огибающие — в баллистике, не отдельной главой
- Кинематика волн — **не включена**

**Со звёздочкой ⭐ внутри разделов:**

## Сборка курса из базы

**Принцип:** база хранит полную картину (все уровни, весь аппарат). Курс — это проекция базы на конкретную аудиторию с конкретным набором доступных инструментов. Драфт текста курса собирается автоматически по правилам профиля.

**Профиль курса** — файл-спецификация, задающий:

| Поле | Пример |
|------|--------|
| Целевая аудитория | 9 класс, уровень 1 |
| Доступный аппарат | конечные разности — да; производная — нет; интеграл — нет |
| Правила подстановки | производная → $\Delta x / \Delta t$ при малом $\Delta t$, с оговоркой что $\Delta t \ll T_{\text{хар}}$ |
| Разделы и порядок | 01, 02, 03 (без 00б, 00в) |
| Исключения | секции ⭐ не включать |

**Механизм сборки:**
1. По профилю выбираются нужные заметки и секции (исключая ⭐ и разделы вне списка)
2. Применяются правила подстановки (замена формул и формулировок)
3. Генерируется драфт (Markdown или LaTeX), который затем редактируется вручную

**Статус:** архитектурное решение принято. Формат профиля и скрипт сборки — реализация позже.

## Система обозначений (conventions/notation.md)

**Решение принято 2026-05-16.** Полный реестр — в `conventions/notation.md`.

**Ключевые конвенции:**
- Свободные векторы — прямой полужирный: $\mathbf{v}$
- Направленные отрезки — стрелка: $\overrightarrow{AB}$
- Модуль вектора — $|\mathbf{v}|$ или курсив $v$
- Единичные векторы (все) — $\hat{\mathbf{e}}$ с индексом: $\hat{\mathbf{e}}_x$, $\hat{\mathbf{e}}_r$, $\hat{\mathbf{e}}_\tau$
- Система координат — $OXY$ / $S$ / набор ортов
- При письме от руки — все векторы стрелкой или чертой сверху (дисклеймер в тексте)

**Координаты и проекции:**
- Координаты вектора — столбец: $\mathbf{v} = (v_x, v_y, v_z)^\top$
- Проекция на ось — нижний индекс: $v_x$, $r_y$, $F_r$, $v_\varphi$

**Пропагация:** в заметках используем `\mathbf{}`, `\hat{\mathbf{e}}_{}`, `\overrightarrow{}`. При смене конвенции — batch-replace. В книге — семантические макросы `\fvec{}`, `\unitv{}`, `\segvec{}`.

## Протокол координации чатов

**Принцип:** архитектуру обсуждаем в одном чате (архитектурном). Заметки по темам пишут отдельные чаты. Каждый тематический чат получает промпт от архитектурного.

**Потоки информации:**

```
┌─────────────────────┐        промпт          ┌──────────────────────┐
│  Архитектурный чат  │ ──────────────────────► │  Тематический чат    │
│  (этот)             │                         │  (пишет заметки)     │
│                     │ ◄────────────────────── │                      │
└─────────────────────┘   context/current-work  └──────────────────────┘
```

**Входящая информация (→ тематический чат):**
- `CLAUDE.md` — правила оформления (авто-загрузка)
- `context/decisions.md` — архитектура и конвенции
- `context/current-work.md` — что сделано, что в работе

**Исходящая информация (тематический чат →):**
- Тематический чат обновляет `context/current-work.md` после каждого значимого продвижения
- Архитектурный чат читает `current-work.md` чтобы узнать о прогрессе

**Типы тематических чатов:**
1. Работа над конкретным разделом (написание заметок)
2. Техническое решение (скрипт, конфигурация, автоматизация)

## Семантические связи между заметками

**Принцип:** структура базы — граф (не линейная цепочка). Заметка может иметь несколько предшественников и несколько продолжений. Связи типизированы по смыслу.

**Реализация — два уровня:**

### 1. Frontmatter (машиночитаемое)

```yaml
relations:
  requires:
    - малые-величины
    - ряды-и-разложения
  leads_to:
    - производная
    - интеграл
```

### 2. Секция «Связи» в теле заметки (для читателя)

```markdown
## Связи

- ← [[Малые величины]] — мотивация: "что если Δx → 0?"
- ← [[ряды-и-разложения]] — инструмент: разложение Тейлора
- → [[Производная]] — предел отношения приращений = производная
- → [[Понятие об интеграле]] — обратная операция
```

### Типы связей

| Тип | Смысл | Пример |
|-----|-------|--------|
| `requires` | Нужно понимать до | предел ← малые-величины |
| `leads_to` | Логически продолжается в | предел → производная |
| `generalizes` | Обобщает | криволинейное обобщает одномерное |
| `applies` | Применяет метод | баллистика применяет векторное-произведение |
| `alternative` | Другой подход к тому же | полярные ↔ декартовы |
| `used_in` | Матаппарат используется в физическом контексте | тригонометрия → баллистика, колебания |

**Назначение:**
- Frontmatter — автомат��ческое построение графа зависимостей, проверка порядка при сборке курса
- Секция «Связи» — мост для читателя: мотивация перехода между темами (почему из X следует изучить Y)

## Структура математического приложения в книге

Математический материал разделён на три уровня сложности:

**Базовая математика** (глава «Математическое введение»):
- Малые величины, оценка точности, геометрия малых углов, ряды и разложения, предел
- Векторы-основы (без скалярного и векторного произведений)
- Геометрический смысл интеграла (только идея площади под кривой, без вычислений)

**Математика следующего уровня** (вводится по мере нужды в физических разделах):
- Дифференцирование, интегрирование
- Скалярное и векторное произведения
- Дифференцирование векторов

**Для продвинутых разделов** (кинематика механизмов, движение в центральном поле и т.п.):
- ОДУ (раздел 00б)
- Дифференциальная геометрия кривых (раздел 00в)

**Обратная синхронизация LaTeX → Wiki:** вручную по запросу.
