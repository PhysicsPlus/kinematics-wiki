## Концепция базы знаний

**База знаний — энциклопедия концепций, а не учебник.**
Учебник (пособие) будет собираться из базы позже: выбором нужных заметок, выстраиванием последовательности, добавлением педагогических переходов. Сейчас важна точность и полнота, а не доступность.

**Три уровня читателей:**

| Уровень | Маркер | Аудитория | Математический аппарат |
|---------|--------|-----------|------------------------|
| **1 — Базовый** | `## ☆` | Школьник 9 класса | Конечные разности, геометрия, площадь под кривой. Без пределов, производных, интегралов. |
| **2 — Основной** | `## ★` | 10–11 класс | Производная, базовое интегрирование, предел. |
| **3 — Продвинутый** | `## ⭐` | Олимпиадники 11 класса | Уверенный анализ, простейшие ОДУ, дифференциальная геометрия. |

## Конвертация Obsidian → LaTeX

Заметки базы конвертируются в главы книги по следующим правилам:

**Иерархия заголовков:**

| Obsidian | LaTeX | Пояснение |
|----------|-------|-----------|
| Раздел (папка с заметками) | `\chapter{...}` | Одна глава на раздел |
| `# Название заметки` | `\section{Название}` | Одна заметка = одна секция |
| `## ☆ Подтема` | `\subsection{Подтема}` | Подраздел внутри секции |
| `## ★ Подтема` | `\subsection{Подтема}` | Подраздел внутри секции |
| `## ⭐ Подтема` | `\subsection{Подтема}` внутри `\begin{remark}...\end{remark}` | Фиолетовая полоса слева |
| `## Связи и продолжения` | пропустить | В книге не нужно |

**Прочие преобразования:**

| Obsidian | LaTeX (книга) | Комментарий |
|----------|---------------|-------------|
| `$$...$$` | `\[...\]` | |
| `> [!definition]` | `\begin{definition}...\end{definition}` | индиго полоса слева |
| `> [!example] Заголовок` | `\begin{example}{Заголовок}...\end{example}` | |
| `[[Имя заметки]]` | убрать | |
| `**термин**` | `\textbf{термин}` | |
| Frontmatter YAML | убрать | |
| Аннотация заметки | вводный абзац секции или убрать | |
| `\mathbf{v}` | `\mbfv` | unicode-math |
| `\mathbf{r}` | `\mbfr` | unicode-math |
| `\boldsymbol{\omega}` | `\mbfomega` | unicode-math |
| `\boldsymbol{\Omega}` | `\mbfOmega` | unicode-math |
| `\hat{\mathbf{e}}_x` | `\hat{\mbfe}_x` | unicode-math |
| `\overrightarrow{AB}` | `\vv{AB}` | esvect |

Семантических макросов (`\fvec`, `\unitv`, `\segvec`) не используем — пишем напрямую.
Полная таблица замен — в `conventions/notation.md`.

**Среды tcolorbox** (определяются в `chap.sty`):
- `definition` — для `[!definition]`, индиго полоса слева
- `example` — для `[!example]`, принимает заголовок: `\begin{example}{Заголовок}`
- `remark` — для секций ⭐, фиолетовая полоса слева
- `theorem`, `proof`, `note` — при необходимости

**Что не конвертируется автоматически:**
- Секция `## Связи и продолжения` — убирается (в книге нет гиперссылок между главами)
- Frontmatter YAML — убирается
- Аннотация заметки — может стать вводным абзацем главы или убирается

**Статус:** преобразования описаны, скрипт автоматической конвертации не реализован. Конвертация выполняется вручную или через промпт Claude.

## Сборка курса из базы (профили)

**Принцип:** база хранит полную картину (все уровни, весь аппарат). Курс — это проекция базы на конкретную аудиторию. Драфт текста курса собирается по правилам профиля.

**Профиль курса** — файл-спецификация:

| Поле | Пример |
|------|--------|
| Целевая аудитория | 9 класс, уровень 1 |
| Доступный аппарат | конечные разности — да; производная — нет; интеграл — нет |
| Правила подстановки | производная → $\Delta x / \Delta t$ при малом $\Delta t$ |
| Разделы и порядок | 01, 02, 03 (без 00б, 00в) |
| Исключения | секции ⭐ не включать |

**Механизм сборки:**
1. По профилю выбираются нужные заметки и секции
2. Применяются правила подстановки (замена формул и формулировок)
3. Генерируется драфт (LaTeX), который затем редактируется вручную

**Статус:** архитектурное решение принято. Формат профиля и скрипт сборки — реализация позже.

## Семантические связи между заметками

**Реализация — два уровня:**

**1. Frontmatter (машиночитаемое):**

```yaml
relations:
  requires:
    - малые-величины
  leads_to:
    - производная
```

**2. Секция «Связи и продолжения» (для читателя):**

```markdown
## Связи и продолжения

Абзац-переход.

- ← [[Малые величины]] — мотивация: "что если Δx → 0?"
- → [[Производная]] — предел отношения приращений = производная
```

**Типы связей:**

| Тип | Смысл | Пример |
|-----|-------|--------|
| `requires` | Нужно понимать до | предел ← малые-величины |
| `leads_to` | Логически продолжается в | предел → производная |
| `generalizes` | Обобщает | криволинейное обобщает одномерное |
| `applies` | Применяет метод | баллистика применяет векторное-произведение |
| `alternative` | Другой подход к тому же | полярные ↔ декартовы |
| `used_in` | Матаппарат используется в физике | тригонометрия → баллистика |
