# Workflow: полная проверка заметки

Этот workflow проверяет одну заметку на соответствие всем правилам.
Правила: `.claude/memory/rules_cached.md`

## Вход

Путь к файлу заметки.

## Чеклист

### A. Frontmatter
- [ ] `status`, `tags`, `level`, `sources`, `latex_packages` заполнены
- [ ] `level` соответствует маркерам (☆→1, ★→2, ⭐→3)
- [ ] `relations.requires` и `relations.leads_to` актуальны

### B. Структура
- [ ] Аннотация (первый абзац после `# Заголовок`)
- [ ] Секции `## ☆/★/⭐` с маркерами уровней
- [ ] Финальная секция `## Связи и продолжения` с абзацем-переходом + список ссылок
- [ ] Нет `###` заголовков
- [ ] Нет `---` горизонтальных разделителей
- [ ] Нет блока `## LaTeX-код`

### C. LaTeX-формулы
- [ ] Дроби: `\frac{}{}` (не косая, кроме показателей степени)
- [ ] Интегралы: `\int\limits_` (не `\int_`)
- [ ] Нет `\dfrac`, `\displaystyle`, `\boxed{}`
- [ ] Тригонометрия: `\operatorname{tg}` (не `\tan`)
- [ ] Векторы: `\mathbf{}` (не `\vec{}`)
- [ ] Числа: запятая `{,}` (не точка)
- [ ] Единицы: через `~` или `\,`
- [ ] `$$...$$` без пустых строк вокруг

### D. Callout-блоки
- [ ] Определения → `[!definition]`
- [ ] Inline-примеры → `[!example]`
- [ ] В `[!example]` нет жирного текста
- [ ] Длинные примеры вынесены в `examples/`

### E. Текст
- [ ] Жирный только для нового термина
- [ ] Каждая нетривиальная формула объяснена
- [ ] `[[Ссылки]]` ведут на существующие файлы

## Выход

```
СТАТУС: OK | N ошибок
[строка N] ОШИБКА: описание → ИСПРАВИТЬ: предложение
```
