# Workflow: проверка раздела Математика

Папка: `concepts/00-Математика, основы/`
Правила: `.claude/memory/rules_cached.md`

## Шаг 1: Сканирование на ошибки LaTeX

Для каждого файла проверить:

1. **Косые дроби.** Найти `/` внутри `$...$` и `$$...$$`. Ошибка, если `/` не в показателе степени `^{...}`. Заменить на `\frac{}{}`.
2. **Интегралы.** Найти `\int_` без `\limits`. Заменить на `\int\limits_`.
3. **\dfrac / \displaystyle.** Найти и заменить на `\frac`.
4. **\vec{}.** Найти и заменить на `\mathbf{}`.
5. **tan/cot/arctan.** Заменить на `\operatorname{tg}` / `\operatorname{ctg}` / `\operatorname{arctg}`.
6. **Десятичная точка.** `0.5` → `0{,}5`.

## Шаг 2: Переделка примеров в [!example]

Найти все строки, начинающиеся с:
- `Пример:`, `Пример 1:`, `Пример 2:` и т.д.
- `**Название.** Текст задачи`
- `Найдём ...`, `Вычислить ...`, `Найти ...` в начале абзаца (если это отдельный пример)

Переделать в:
```markdown
> [!example] Заголовок
> Условие задачи.
>
> Решение и ответ.
```

Правила:
- Жирный текст внутри [!example] не используется
- Заголовок — краткий (5-10 слов)
- Между условием и решением — пустая строка `>`

## Шаг 3: Проверка определений

- Все определения должны быть в `[!definition]` callout
- Строки вида `**Определение.**` или `Определение:` → переделать в callout

## Шаг 4: Структура

- Нет `###` заголовков
- Нет `---` разделителей
- Жирный только для новых терминов
- Есть `## Связи и продолжения` в конце
- Маркеры уровней: `☆`, `★`, `⭐`
