# Правила работы с базой знаний kinematics-wiki

Этот файл автоматически загружается Claude Code. Все правила обязательны при написании и редактировании заметок.

---

## Общая концепция

- База знаний — энциклопедия концепций и методов, **не учебник**. Учебник будет собираться из базы позже.
- Стиль заметок — точный, как в хорошем справочнике или теормехе.
- Читатель базы не знает ничего, кроме квадратного уравнения — но при этом база обслуживает разных читателей: от школьника 9 класса до студента, знающего теормех.

### Уровни сложности

- **Уровень 1** — школьник 9 класса среднего уровня
- **Уровень 2** — старшеклассник физ-мат класса
- **Уровень 3 (⭐)** — студент, знающий теормех

Заметка может содержать несколько уровней: базовая часть доступна уровню 1–2, секция `## ⭐ ...` — уровню 3.

## Правила оформления формул (LaTeX)

### Дроби

- **Никогда** не писать косые дроби `a/b` в формулах — только в крайнем случае (например, в тексте: «при $v/c \ll 1$»).
- Всегда использовать `\frac{числитель}{знаменатель}`.
- **Не использовать** `\dfrac` и `\displaystyle` — только `\frac`.

```tex
% Правильно:
\frac{v^2}{R}

% Неправильно:
v^2/R
\dfrac{v^2}{R}
{\displaystyle \frac{v^2}{R}}
```

### Рамки и выделение

- **Не обводить** формулы рамкой (`\boxed{}`). Исключение — если пользователь явно просит.
- Для выделения ключевых формул использовать подписи или отдельный подраздел, но не рамки.

### Выключные формулы

- Использовать `$$...$$` для выключных формул (Obsidian рендерит).
- Строчные формулы — `$...$`.
- В секции `## LaTeX-код` использовать оригинальный синтаксис книги (`\[...\]`, `equation`, `align`).

### Векторы и специальные обозначения

- Пакеты: `physics` и `unicode-math`.
- Векторы: `\vec{v}` или `\vb{v}` (physics-пакет).
- Орты: `\hat{e}` или `\vu{e}`.
- Производная по времени: `\dot{x}`, `\ddot{x}`.
- Дифференциал: `\dd{t}` (physics) или `\mathrm{d}t`.

### Прочие правила оформления

- Запятая в числах — запятая, не точка: `0{,}5`, не `0.5` (русская типографика).
- Единицы измерения через `~` или `\,`: `9{,}8~\text{м/с}^2`.
- Многоточие: `\cdots` (в формулах), `\ldots` (в тексте).

## Правила написания

- Писать подробно, с примерами — «разжёвывать до мельчайших подробностей, не теряя строгости».
- Каждое нетривиальное утверждение сопровождать примером или выводом.
- Формулы, которые «просто написаны» без объяснения — плохой тон. Объяснять откуда берётся каждая формула.
- Таблицы — хорошо, особенно для числовых значений и сравнений.
- Физические примеры предпочтительнее абстрактных.

## Технические правила

- После написания/редактирования заметок делать `git add . && git commit && git push`.
- Сообщение коммита: кратко что добавлено/изменено.
- PDF учебников в `sources/` не добавлять в git (уже в `.gitignore`).
- LaTeX-файлы авторских разработок — добавлять в git (они текстовые).
