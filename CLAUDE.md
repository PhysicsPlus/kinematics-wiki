# Правила работы с базой знаний kinematics-wiki

Этот файл автоматически загружается Claude Code. Все правила обязательны при написании и редактировании заметок.

---

## Общая концепция

- База знаний — энциклопедия концепций и методов, **не учебник**. Учебник будет собираться из базы позже.
- Стиль заметок — точный, как в хорошем справочнике или теормехе.
- Читатель базы не знает ничего, кроме квадратного уравнения — но при этом база обслуживает разных читателей: от школьника 9 класса до студента, знающего теормех.

### Уровни сложности

| Уровень | Аудитория | Математический аппарат |
|---------|-----------|------------------------|
| **1 — Базовый** | Школьник 9 класса | Конечные разности, геометрия, площадь под кривой. Без пределов, производных, интегралов |
| **2 — Основной** | 10–11 класс | Производная, базовое интегрирование, предел как строгое понятие |
| **3 — Продвинутый** | Олимпиадники 11 класса | Уверенное дифференцирование, интегрирование, простейшие ОДУ, дифференциальная геометрия |

### Маркеры уровней в тексте заметки

- **`## ☆ ...`** — уровень 1 (Базовый). Доступен школьнику 9 класса.
- **`## ★ ...`** — уровень 2 (Основной). Для тех, кто знает производную/интеграл.
- **`## ⭐ ...`** — уровень 3 (Продвинутый). Для олимпиадников и студентов теормеха.

Не каждая заметка содержит все три блока — только те уровни, которые релевантны теме.

Пример структуры заметки про мгновенную скорость:
```
## ☆ Мгновенная скорость (базовый уровень)
  v_x = Δx/Δt при малом Δt

## ★ Мгновенная скорость через предел
  v_x = lim(Δt→0) Δx/Δt = dx/dt

## ⭐ Скорость как вектор в n-мерном пространстве
  ...
```

### Поле level во frontmatter

```yaml
level: 1        # только базовый
level: 2        # только основной
level: 3        # только продвинутый
level: 1-2      # базовый + основной
level: 2-3      # основной + продвинутый
level: 1-3      # все уровни
```

## Правила оформления формул (LaTeX)

### Дроби

- Везде использовать `\frac{}{}` — никаких косых дробей.
- **Исключение:** в показателях степени косая дробь допустима, если `\frac` делает запись громоздкой: `$e^{-t^2/2}$`, `$x^{1/2}$`, `$x^{-3/2}$`.
- **Не использовать** `\dfrac` и `\displaystyle` — только `\frac`.

```tex
% Везде \frac — и в выключных, и в строчных формулах:
a_n = \frac{v^2}{R}
$\frac{n(n-1)}{2}x^2$

% Исключение — косая только в показателях степени:
$e^{-t^2/2}$, $x^{1/2}$, $x^{-3/2}$
```

### Интегралы

- Пределы интегрирования — всегда через `\limits`: `\int\limits_a^b`
- Без `\limits` пределы ставятся сбоку (как в строчных формулах) — это не наш стиль.

```tex
% Правильно:
\int\limits_0^T f(t)\,\mathrm{d}t

% Неправильно:
\int_0^T f(t)\,\mathrm{d}t
```

### Перенос длинных формул

- Если формула не помещается в одну строку, переносить по знаку равенства: строка заканчивается на `=`, следующая начинается с `=`.
- Для переноса использовать окружение `aligned` внутри `$$`.
- Между строками с крупными объектами (матрицами, дробями) добавлять вертикальный промежуток: `\\[8pt]`.

```tex
% Перенос формулы по =:
$$
\begin{aligned}
\text{длинное выражение} &= \\
&= \text{результат}
\end{aligned}
$$
```

### Несколько формул в одном блоке

Две формулы, разделённые `\qquad` в одной строке, нужно разбивать на отдельные строки. Выбор окружения зависит от того, нужно ли выравнивание:

**`gathered`** — несколько независимых уравнений, выровненных по центру (каждое на своей строке):
```tex
$$
\begin{gathered}
\sin x \approx x \\
\cos x \approx 1 - \frac{x^2}{2}
\end{gathered}
$$
```

**`aligned` с `&` в начале строки** — список уравнений, выровненных по левому краю:
```tex
$$
\begin{aligned}
&e^x = 1 + x + \frac{x^2}{2!} + \cdots \\[8pt]
&\sin x = x - \frac{x^3}{6} + \cdots
\end{aligned}
$$
```

**`aligned` с `&=`** — цепочка равенств с переносом (см. раздел «Перенос длинных формул»).

Между строками с крупными объектами (матрицами, длинными дробями) добавлять вертикальный промежуток `\\[8pt]`. Для обычных строк достаточно `\\`.

### Рамки и выделение

- **Не обводить** формулы рамкой (`\boxed{}`). Исключение — если пользователь явно просит.
- Для выделения ключевых формул использовать подписи или отдельный подраздел, но не рамки.

### Выключные формулы

- Использовать `$$...$$` для выключных формул (Obsidian рендерит).
- **Не оставлять пустых строк** перед `$$` и после `$$` — иначе Obsidian создаёт лишние вертикальные отступы. Правильно:
  ```markdown
  Текст перед формулой:
  $$
  a = \frac{v^2}{R}
  $$
  Текст после формулы.
  ```
- Строчные формулы — `$...$`.
- В секции `## LaTeX-код` использовать оригинальный синтаксис книги (`\[...\]`, `equation`, `align`).

### Тригонометрические функции

- Русская нотация: **tg**, **ctg**, **arctg**, **arcctg** (не tan, cot, arctan, arccot).
- В формулах: `\operatorname{tg}`, `\operatorname{ctg}`, `\operatorname{arctg}`, `\operatorname{arcctg}`.
- В LaTeX-книге определить в преамбуле: `\DeclareMathOperator{\tg}{tg}` и аналогично для остальных.

### Векторы и специальные обозначения

- Пакеты: `physics` и `unicode-math`.
- Векторы **латинскими** буквами: `\mathbf{v}`, `\mathbf{r}`, `\mathbf{a}` и т.д. Стрелки (`\vec{}`) не использовать.
- Векторы **греческими** буквами: `\boldsymbol{\Omega}`, `\boldsymbol{\omega}` и т.д.
- Орты: `\hat{\mathbf{e}}_x`, `\hat{\mathbf{e}}_y`, `\hat{\mathbf{e}}_z`.
- Производная по времени: `\dot{x}`, `\ddot{x}`.
- Дифференциал: `\dd{t}` (physics) или `\mathrm{d}t`.

### Прочие правила оформления

- Запятая в числах — запятая, не точка: `0{,}5`, не `0.5` (русская типографика).
- Единицы измерения через `~` или `\,`: `9{,}8~\text{м/с}^2`.
- Многоточие: `\cdots` (в формулах), `\ldots` (в тексте).

```markdown
# Название

Аннотация: что излагается в заметке, какое место занимает в структуре базы,
с какими заметками связана логически ([[ссылки]]).

## ☆ / ## ★ / ## ⭐  Содержательные блоки по уровням
(только те уровни, которые релевантны теме)

## Связи и продолжения

Абзац, объясняющий логику переходов: где мы находимся после этой заметки
и почему каждая следующая тема является естественным продолжением.

- → [[заметка]] — одна строка: что там
```

### Правила оформления блоков

- Заголовки второго уровня (`##`) — для смысловых разделов. Заголовки третьего уровня (`###`) не использовать: если нужно выделить подтему — абзац с вводным предложением.
- Жирный шрифт — только для термина, который вводится впервые в этом месте. Не выделять жирным выводы, пояснения, подзаголовки внутри абзацев.
- Примеры бывают двух видов: короткие разбираются прямо по ходу текста; длинные оформляются отдельной заметкой, в основной — только ссылка.
- Блок `## LaTeX-код` не нужен.

### Определения

Ключевые определения и законы оформляются как Obsidian-callout:

```markdown
> [!definition]
> **Название (уточнение).** Формулировка определения.
> $$\text{формула}$$
> где $x$ — ..., $y$ — ...
```

Callout-определение используется для:
- точных математических определений понятий (скорость, ускорение, СО…)
- ключевых законов (закон сложения скоростей, формула Кориолиса…)

Не каждая формула является определением. Выводы, следствия и частные случаи — обычный текст.

### Примеры

Короткие inline-примеры внутри концептуальных блоков оформляются callout'ом:

```markdown
> [!example] Краткий заголовок
> Формулировка: что найти или вычислить.
>
> Решение: вывод и ответ.
```

Длинные примеры — отдельный файл `examples/*/пример-*.md`, в заметке только ссылка `[[...]]`.
Жирный текст внутри `[!example]` не используется.

## Правила написания

- Писать подробно, с примерами — «разжёвывать до мельчайших подробностей, не теряя строгости».
- Каждое нетривиальное утверждение сопровождать примером или выводом.
- Формулы, которые «просто написаны» без объяснения — плохой тон. Объяснять откуда берётся каждая формула.
- Таблицы — хорошо, особенно для числовых значений и сравнений.
- Физические примеры предпочтительнее абстрактных.

## Рисунки

### Хранение

Все рисунки хранятся в `figures/`, организованы по темам:

```
figures/
  01-basics/
    velocity-graph.svg      ← для Obsidian
    velocity-graph.eps      ← исходник Illustrator / для LaTeX
  02-vectors/
    vector-addition.svg
    vector-addition.eps
    vector-addition.py      ← исходник Python
  photos/
    setup-photo.jpg
```

| Источник | Файлы в figures/ |
|----------|-----------------|
| Adobe Illustrator | `.eps` (исходник) + `.svg` (для Obsidian) |
| Python-скрипт | `.py` (исходник) + `.svg` (для Obsidian) |
| Фотография | `.jpg` |

### Вставка в заметки Obsidian

Ссылаться только на `.svg`:
```markdown
![[figures/01-basics/velocity-graph.svg]]
```

### Публикация в LaTeX

При команде опубликовать часть базы — копируем нужные рисунки в подпапки LaTeX-проекта:
- Illustrator: копируем `.eps`
- Python: генерируем `.pdf` из `.py` и копируем
- Фото: копируем `.jpg`
