### Архитектура: три уровня сложности

**Принято решение:** три уровня вместо прежних «уровень 1–2 + ⭐»:

| Уровень | Аудитория | Аппарат | Маркер секции |
|---------|-----------|---------|---------------|
| 1 — Базовый | 9 класс | Конечные разности, геометрия | (основной текст) |
| 2 — Основной | 10–11 класс | Производная, интеграл, предел | `## ◆ ...` |
| 3 — Продвинутый | Олимпиадники | ОДУ, дифгеометрия | `## ⭐ ...` |

**Ключевой принцип:** основной текст заметки объясняет понятия через конечные разности и геометрию (без пределов). Блок `## ◆` добавляет строгое определение через предел/производную. Блок `## ⭐` — для продвинутых.

### Семантические связи между заметками

**Решение:** заметки связаны типизированными связями (граф, не линейная цепочка). Реализация на двух уровнях:
- **Frontmatter** (`relations: requires / leads_to / generalizes / applies / alternative`) — для автоматического построения графа и сборки курса
- **Секция «Связи»** в теле заметки — мост для читателя с объяснением, почему из X следует перейти к Y

Типы: `requires` (предпосылка), `leads_to` (продолжение), `generalizes` (обобщение), `applies` (применение метода), `alternative` (другой подход).
